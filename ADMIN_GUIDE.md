## Archive 관리

### 프로젝트 추가
1. **Archive** 탭에서 **"+ 새 프로젝트 추가"** 클릭
2. 타입 선택: 이미지, 비디오, YouTube, 텍스트, 웹사이트
3. 파일 업로드 또는 URL 입력
4. 제목, 태그, 연도, 설명 입력
5. **💾 저장** 클릭 → 즉시 반영

### 프로젝트 수정
1. 프로젝트 목록에서 **"수정"** 버튼 클릭
2. 정보 수정 후 **💾 저장**

### 프로젝트 삭제
1. 프로젝트 목록에서 **"삭제"** 버튼 클릭
2. 확인 대화상자에서 확인

### 파일 업로드
- **지원 형식**: JPEG, PNG, GIF, WebP, MP4, WebM
- **최대 크기**: 50MB
- **저장 위치**: `public/uploads/`

## 방명록 관리

### 방명록 수정
1. **방명록** 탭 클릭
2. 항목에 마우스를 올리면 **"수정"** 버튼 표시
3. 이름과 메시지 수정 후 **"저장"** 클릭

### 방명록 삭제
1. 항목에 마우스를 올리면 **"삭제"** 버튼 표시
2. 확인 대화상자에서 확인

## 문제 해결

### 저장 버튼이 비활성화됨
- **원인**: URL 또는 제목이 비어있음
- **해결**: 필수 필드 입력 확인

### 파일 업로드 실패
- **원인 1**: 백엔드 서버가 실행되지 않음
  - **해결**: `npm run server` 실행
- **원인 2**: 파일 크기 초과
  - **해결**: 50MB 이하 파일 사용
- **원인 3**: 지원하지 않는 형식
  - **해결**: JPEG, PNG, MP4 등 지원 형식 사용

### 이미지가 표시되지 않음
1. 백엔드 서버 재시작 (`Ctrl + C` → `npm run server`)
2. 브라우저 강력 새로고침 (`Ctrl + Shift + R`)
3. 개발자 도구 (F12) → Network 탭에서 이미지 요청 확인

### 데이터가 반영되지 않음
- 브라우저 캐시 삭제: `Ctrl + Shift + R`
- 개발자 도구 → Network 탭 → "Disable cache" 체크
