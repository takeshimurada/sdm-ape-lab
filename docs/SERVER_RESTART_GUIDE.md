# 🔄 개발 서버 재시작 가이드

## 📋 재시작이 필요한 이유

### 1. **설정 파일 변경 시**
- `vite.config.ts` 파일을 수정했을 때 (포트, 프록시 등)
- `package.json`의 스크립트를 변경했을 때
- 환경 변수 파일을 변경했을 때

**이유**: Vite는 시작할 때 설정 파일을 한 번만 읽습니다. 실행 중에는 자동으로 다시 읽지 않습니다.

### 2. **포트 충돌 해결**
- 다른 프로그램이 같은 포트를 사용하고 있을 때
- 예: WordPress가 포트 3000을 사용 → Vite를 포트 5173으로 변경

**이유**: 포트는 서버가 시작될 때 할당됩니다. 실행 중에는 변경할 수 없습니다.

### 3. **의존성 설치 후**
- `npm install`로 새 패키지를 설치했을 때
- `package.json`의 의존성을 업데이트했을 때

**이유**: 새로 설치된 패키지는 서버 재시작 후에만 사용할 수 있습니다.

### 4. **에러 발생 시**
- 서버가 응답하지 않을 때
- 빌드 에러가 발생했을 때
- 메모리 누수로 인한 성능 저하

**이유**: 재시작하면 모든 상태가 초기화되어 문제가 해결될 수 있습니다.

---

## 🚀 재시작 방법

### 방법 1: 현재 터미널에서 재시작 (가장 간단)

**1단계: 서버 중지**
- 터미널에서 `Ctrl + C` 누르기
- 서버가 종료될 때까지 대기 (보통 1-2초)

**2단계: 서버 다시 시작**
```powershell
npm run dev
```

또는 전체 스택 실행:
```powershell
npm run dev:full
```

---

### 방법 2: 두 개의 서버가 실행 중일 때

**백엔드 서버 (포트 3001)와 프론트엔드 서버 (포트 5173)가 따로 실행 중이라면:**

#### 백엔드만 재시작:
1. 백엔드 서버 터미널에서 `Ctrl + C`
2. 다시 실행:
```powershell
npm run server
```

#### 프론트엔드만 재시작:
1. 프론트엔드 서버 터미널에서 `Ctrl + C`
2. 다시 실행:
```powershell
npm run dev
```

#### 둘 다 재시작:
1. 각 터미널에서 `Ctrl + C`로 모두 종료
2. 새 터미널에서:
```powershell
npm run dev:full
```

---

### 방법 3: 프로세스 강제 종료 후 재시작

**서버가 응답하지 않을 때 사용:**

#### Windows PowerShell:
```powershell
# 포트 5173 사용 중인 프로세스 찾기
netstat -ano | findstr :5173

# PID 확인 후 종료 (예: PID가 12345인 경우)
taskkill /PID 12345 /F

# 포트 3001 사용 중인 프로세스 찾기
netstat -ano | findstr :3001

# PID 확인 후 종료
taskkill /PID [PID번호] /F

# 재시작
npm run dev:full
```

## 📌 현재 상황 (포트 변경 후)

### 변경 사항:
- **이전**: 포트 3000 (WordPress와 충돌)
- **현재**: 포트 5173

### 재시작 절차:

1. **현재 실행 중인 서버 종료**
   - 터미널에서 `Ctrl + C`

2. **서버 다시 시작**
   ```powershell
   npm run dev:full
   ```

3. **브라우저에서 접속**
   ```
   http://localhost:5173
   ```

## ✅ 재시작 확인 방법

### 1. 터미널 메시지 확인
정상적으로 시작되면 다음과 같은 메시지가 표시됩니다:

```
VITE v6.x.x  ready in xxx ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
```

### 2. 브라우저에서 접속 테스트
- http://localhost:5173 접속
- 페이지가 정상적으로 로드되는지 확인

### 3. 개발자 도구 확인
- `F12` 키로 개발자 도구 열기
- `Console` 탭에서 에러가 없는지 확인

## 🔍 문제 해결

### ❌ "포트가 이미 사용 중입니다" 에러

**해결책:**
1. 다른 프로그램이 포트를 사용 중인지 확인
2. 해당 프로세스 종료
3. 또는 `vite.config.ts`에서 다른 포트로 변경

### ❌ "Cannot find module" 에러

**해결책:**
```powershell
# 의존성 재설치
npm install

# 서버 재시작
npm run dev
```

### ❌ 서버가 시작되지 않아요

**해결책:**
1. 터미널을 완전히 닫고 새로 열기
2. 프로젝트 디렉토리로 이동 확인
3. `npm install` 실행
4. 다시 `npm run dev` 실행

## 💡 팁

### 자동 재시작이 필요한 경우
- **코드 변경**: 자동으로 반영됨 (재시작 불필요)
- **설정 변경**: 재시작 필요
- **의존성 변경**: 재시작 필요

### 빠른 재시작 단축키
1. `Ctrl + C` (서버 중지)
2. `↑` (이전 명령어 불러오기)
3. `Enter` (다시 실행)
